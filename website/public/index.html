Diagrams your agents can edit

Agentic Mermaid turns Mermaid diagrams into files your docs, pull requests, and terminals can use: SVG and PNG for pages, ASCII and Unicode for text surfaces. It also gives coding agents a safe editing path for existing diagrams, so they can parse the source, change the intended node or edge, verify the result, and hand back Mermaid you can review instead of redrawing the whole diagram from scratch.

Rules:
- Do not call the website as a render API.
- Parse with parseMermaid first.
- Narrow with the matching as* helper.
- Use typed mutate operations when the family supports them.
- Run verifyMermaid before serialize, render, commit, or return.
- If structural warnings remain after two attempts, return the warnings and ask for review.

How the edit loop stays safe

  1. Parse. Start from Mermaid source, not a screenshot or regenerated guess.
  2. Narrow. Resolve the request to a family-specific surface like asFlowchart, asSequence, or asGantt.
  3. Mutate. Change the intended node, edge, task, relation, or event while unmodeled syntax remains preserved source.
  4. Verify. Read structural, geometric, and lint tiers before serializing or returning artifacts.

Local setup

Self-hosting over stdio is the default path. The website is documentation plus a browser-local editor; it is not a render API and it does not run hosted Code Mode.

$ git clone https://github.com/adewale/beautiful-mermaid
$ cd beautiful-mermaid && bun install && bun run build
$ bun run bin/am.ts --help
$ bun run bin/agentic-mermaid-mcp.ts      # stdio MCP

Claude Code, Cursor, Codex, Gemini CLI, GitHub Copilot, Pi, OpenCode, and generic MCP clients all use the same local command. Client-specific setup is exposed in harnesses.json.

One model, five outputs

Every family renders to vector, raster, and text from one layout, byte-identical across processes. Outputs: SVG, PNG, ASCII, Unicode, and JSON layout, across twelve diagram families.
