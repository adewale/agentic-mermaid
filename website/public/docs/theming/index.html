Theming and styles

A style describes how diagrams look; a colors-only style is a theme.

One primitive covers every look: a style is a partial, composable description of how diagrams render — palette, typography, stroke character, fills. A style that only sets colours is what people call a theme; full looks like hand-drawn, watercolor, or blueprint also change the mark treatment. Styles stack left → right ({ style: ['hand-drawn', 'dracula'] } is hand-drawn geometry in the dracula palette), the optional seed re-rolls styled ink without ever moving layout, and custom styles are plain JSON records. The browser editor exposes both pickers — Style chooses the look, Theme chooses the palette — and SVG output can also inherit CSS variables for live theming.
