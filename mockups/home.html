Diagrams your agents can edit

Render Mermaid to SVG, PNG, ASCII, or Unicode, then parse, verify, and safely mutate it through a typed surface built for agents.

Layout is deterministic: identical input produces identical geometry. Anything the model cannot represent round-trips losslessly through preserved source, so nothing is silently dropped.

Start

Three ways in.

  • Try it in the browser. Paste Mermaid, get SVG/PNG and text output, copy a share link. Open the editor
  • Install locally вҖ” library, CLI, or a local MCP server over stdio:
$ npm i agentic-mermaid
$ npx agentic-mermaid-mcp      # stdio by default; HTTP is opt-in
  • Configure an agent. am init-agent wires it into Claude Code, Cursor, Codex, Gemini CLI, GitHub Copilot, Pi, OpenCode, or any generic MCP client.

One model, four ways out

Every family renders to vector, raster, and text from one layout, byte-identical across processes. Outputs: SVG, PNG, ASCII, Unicode, and JSON layout, across twelve diagram families.

Edit one node without regenerating the diagram

A render-only library makes an agent regenerate the whole diagram to move one node. Here, existing diagrams go parse → narrow → mutate → verify → serialize, and verify returns structured warnings in three tiers plus perceptual quality metrics. An agent reads the tiers and decides whether to fix, re-narrow, or stop and ask a human.

Local by default

Run the library, the am CLI, or a local MCP server over stdio. Code Mode execute stays local, for trusted use only. An optional hosted MCP exposes only render, verify, describe, and structured edits, authless and rate-limited, with no arbitrary code. Self-hosting is the recommended path.
