Agents / Harnesses

Set up your harness

Add the local MCP server over stdio, then run the smoke task to confirm the agent can render, verify, and edit. Each card is generated from harnesses.json.

Recommended

Claude Code

One command registers the stdio server. No HTTP, no token, no account.

Smoke task: “Render workflow.mmd to PNG, then verify it and fix any lint.”

Channels: library · CLI · local MCP · skill

copy
# register the local server
$ claude mcp add agentic-mermaid \
    -- npx agentic-mermaid-mcp
# confirm tools are reachable
$ claude mcp list
✓ agentic-mermaid · render verify describe mutate

Cursor

Add a server entry to ~/.cursor/mcp.json.

"agentic-mermaid": {
  "command": "npx",
  "args": ["agentic-mermaid-mcp"]
}

Smoke: render + verify a flowchart.

Codex

Declare it under [mcp_servers] in config.toml.

[mcp_servers.agentic-mermaid]
command = "npx"
args = ["agentic-mermaid-mcp"]

Smoke: mutate a node, re-verify.

Gemini CLI

Register the stdio server.

$ gemini mcp add \
  agentic-mermaid \
  npx agentic-mermaid-mcp

Smoke: export PNG to a file.

GitHub Copilot

Add to .vscode/mcp.json in the workspace.

"servers": {
 "agentic-mermaid": { ... }
}

Smoke: describe a diagram.

Pi · OpenCode

Both read a stdio MCP entry from their config.

$ npx agentic-mermaid-mcp
# add under mcp: in the config

Smoke: parse → mutate → serialize.

Generic MCP

Any client that speaks MCP over stdio.

$ npx agentic-mermaid-mcp
# transport: stdio (default)
# --transport http to opt in

Smoke: call capabilities.

note

Self-hosting over stdio is the default and the recommended path. The optional hosted MCP is a convenience for clients that cannot self-host: it exposes only render, verify, describe, and structured edits, rate-limited and authless. It never runs Code Mode execute.
