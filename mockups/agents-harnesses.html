Set up your harness

Add the local MCP server over stdio, then run the smoke task to confirm the agent can render, verify, and edit.

Every block below is generated from harnesses.json. Self-hosting over stdio is the default and the recommended path.

Claude Code

One command registers the stdio server. No HTTP, no token, no account.

# register the local server
$ claude mcp add agentic-mermaid -- npx agentic-mermaid-mcp
$ claude mcp list
✓ agentic-mermaid · render verify describe mutate

Smoke task: “Render workflow.mmd to PNG, then verify it and fix any lint.”

Cursor

Add a server entry to ~/.cursor/mcp.json.

"agentic-mermaid": { "command": "npx", "args": ["agentic-mermaid-mcp"] }

Codex

Declare it under [mcp_servers] in config.toml.

[mcp_servers.agentic-mermaid]
command = "npx"
args = ["agentic-mermaid-mcp"]

Gemini CLI

Register the stdio server.

$ gemini mcp add agentic-mermaid npx agentic-mermaid-mcp

GitHub Copilot

Add to .vscode/mcp.json in the workspace.

"servers": { "agentic-mermaid": { "command": "npx", "args": ["agentic-mermaid-mcp"] } }

Pi · OpenCode

Both read a stdio MCP entry from their config.

$ npx agentic-mermaid-mcp     # add under mcp: in the config

Generic MCP

Any client that speaks MCP over stdio.

$ npx agentic-mermaid-mcp           # transport: stdio (default)
$ npx agentic-mermaid-mcp --transport http   # opt in to HTTP
The optional hosted MCP is a convenience for clients that cannot self-host. It exposes only render, verify, describe, and structured edits, rate-limited and authless. It never runs Code Mode execute.
