The diagram-workflow skill

One guided task that wraps the whole edit loop. Describe the change in words; the skill narrows to the right node, mutates, verifies, and hands back a tiered report — no raw tool calls to orchestrate.

This is the curated public skill, agentic-mermaid-diagram-workflow. The live-editor skill stays out of agent onboarding; for an agent editing diagrams, this is the one to reach for.

The shape of a run

Four moves, the same every time.

  1. Describe. “Move the verify node below mutate and label the back-edge warnings.” Plain language, no node IDs.
  2. Narrow. The skill resolves your words to a node or edge and scopes the edit there, so nothing else can move.
  3. Mutate and verify. It applies the change and runs verify; structural, geometric, and lint tiers come back with perceptual metrics.
  4. Report. You get the new render plus the tiers — ship, re-narrow, or stop and ask.

A run, end to end

> describe: label the V→N edge "warnings", move N left of M
✓ narrowed: edge V→N, node N
✓ mutated: 1 edge labelled, 1 node moved
verify:
  tier 1 · structural   0
  tier 2 · geometric    0
  tier 3 · lint         1  (edge label overlaps node M)
  legibility 0.97 · edge crossings 0
→ re-narrowed: nudged the label offset · lint 0 · shipped

Because layout is deterministic, the re-narrow moved only the label — every other node landed exactly where it was, so the agent never re-checks work it already trusted.

When to use it

Reach for the workflow skill to change an existing diagram — the loop is the whole point. To author a diagram from scratch, the plain render tool is enough; for interactive, human-in-the-loop editing, the live editor (kept out of agent onboarding) is the better fit. Wire up a harness

The skill never runs Code Mode execute. It calls render, narrow, mutate, and verify — all available over the local stdio MCP, all deterministic.
