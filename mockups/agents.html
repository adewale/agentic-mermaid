Agentic Mermaid / Agents

A diagram surface built for agents

A render-only library makes an agent regenerate the whole diagram to move one node. Agentic Mermaid exposes a typed surface — parse, narrow, mutate, verify, serialize — so an agent changes only what it means to and leaves the rest untouched.

Two things make that safe: layout is deterministic, so an edit can't silently shift a node that was already correct, and verify answers in structured tiers an agent can act on. Everything below runs locally; the hosted MCP is optional.

The edit loop

Existing diagrams go parse → narrow → mutate → verify → serialize. narrow scopes the edit to one node or edge; mutate applies it; verify returns warnings in three tiers — structural, geometric, lint — plus perceptual quality metrics. An agent reads the tiers and decides whether to ship, re-narrow, or stop and ask a human.

Run it locally

The library, the am CLI, and a local MCP server over stdio all ship in one package. Code Mode execute stays local, for trusted use only.

$ npm i agentic-mermaid
$ npx agentic-mermaid-mcp      # stdio by default; HTTP is opt-in

Each harness — Claude Code, Cursor, Codex, Gemini CLI, Copilot, Pi, OpenCode — registers the stdio server with one line. Set up your harness

The diagram-workflow skill

The consumer skill, agentic-mermaid-diagram-workflow, wraps the edit loop as one guided task: describe the change in words, and the skill narrows, mutates, verifies, and reports back in tiers. See the workflow

Machine-readable context

The same site an agent can read without scraping a page. Each surface is a stable URL:

  • llms.txt — the one-screen orientation: what the tool is, the loop, the limits.
  • agent-instructions.md — the operating guide: when to narrow, how to read a verify result.
  • capabilities.json — formats, families, and the exact tool signatures.
  • agent-manifest.json — the manifest a client discovers the server with.
  • harnesses.json — the per-client setup blocks, the source for the harness page.
  • schemas/ — the JSON Schemas for every tool's input and output.
Self-hosting over stdio is the default and the recommended path. The optional hosted MCP is authless and rate-limited, exposes only render, verify, describe, and structured edits, and never runs Code Mode execute.
